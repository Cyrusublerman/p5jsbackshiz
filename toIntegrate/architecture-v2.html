Displacement Matrix Pipeline v2

Continuous forces · persistent state · mix knobs

Key Shift: Forces, Not Teleportation

Before — instant displacement

Each frame:
  matrix = zeros()         ← rebuilt from scratch
  matrix += gradient × 500 ← teleport 500px
  matrix += sine × 30      ← teleport 30px
  render(base + matrix)

Problems:
  No inertia, no settling
  Magnetism needs separate pass
  Forces fight each other instantly
  Parameters scale as pixels (unintuitive)

After — continuous forces

Each frame:
  matrix *= (1 - damping)       ← decay toward rest
  matrix += gradient × str × dt ← gentle push
  matrix += sine × str × dt     ← gentle push
  matrix += magnetism × dt      ← same pipeline!
  modulate(matrix)              ← steer, scale
  render(base + matrix)

Benefits:
  Forces settle into equilibrium
  Magnetism is just another force
  Everything balances naturally
  Damping prevents runaway
The matrix persists across frames. Forces don't set positions — they push. Damping pulls back toward zero. The equilibrium between force strength and damping determines final displacement. Turning a force off lets the matrix decay back to rest naturally.

Mix Knobs

Every module has a 0–1 mix controlling its contribution. For generators and constraints, mix scales the force added per frame. For modulators, mix is a wet/dry blend on the transform.

CategoryMix = 0Mix = 0.5Mix = 1Formula
Generator Adds nothing Half-strength push Full push matrix += force × mix × dt
Modulator Pass-through 50/50 blend Fully transformed out = lerp(in, transformed, mix)
Constraint No correction Half-strength restoring Full restoring force matrix += correction × mix × dt
Damping No decay (runaway!) Slow return to rest Near-instant decay matrix *= (1 - damping × dt)

Revised Pipeline

Stage 0 — Sources (read-only)
Computed once at image load or per-frame for animated values
VectorFieldMap angle · mag · lum · cosA · sinA · tanCos · tanSin
Line Geometry base positions · perpendiculars · ordering
Time + Origin t, dt, animated origin xy
↓
Stage 1 — Damping
Decay existing displacement toward zero. Without this, forces accumulate without bound. Controls how "fluid" the system feels — high damping = responsive, low = drifty.
Global Damping matrix[l][p] *= (1 − damping × dt)
↓
Stage 2 — Generators (additive forces)
Each computes a force vector and adds it to the matrix, scaled by its mix knob and dt. All purely additive. Zero mix = no contribution.
Fixed Angle mix · direction × lum × amp
Origin Radial mix · unit vec from origin × lum × str
Gradient Push mix · raw grad vector × str
Tangent Push mix · raw tangent vector × str
Flow Field mix · noise direction × str × lum
Sine Waves mix · wave sum × perp dir × amp
↓
Stage 3 — Constraints (restoring forces)
Continuous restoring forces that push lines toward structurally valid positions. Magnetism samples luminance between line pairs and applies a push force proportional to the error between current gap and target gap. Runs every frame. Propagates because pushing one line changes the gaps of its neighbors in the next frame.
Magnetism mix · (targetGap − currentGap) × str × dt
Boundary mix · push inward when near edge
↓
Stage 4 — Modulators (transform in-place)
Operate on the composite vector. Each modulator blends between the unmodified vector and its transformed version using its mix knob. Applied to the accumulated result, not per-generator.
Steer → Gradient mix · rotate direction toward gradient angle
Steer → Tangent mix · rotate direction toward tangent angle
Edge Mag Scale mix · attenuate by gradient strength
Lum Scale mix · attenuate by brightness
↓
Stage 5 — Render
Read final matrix. Draw: curveVertex(base[l][p] + matrix[l][p]). Apply colour.
curveVertex(base + matrix)
↻ matrix persists — next frame starts from Stage 1 (damping)

Magnetism as Continuous Force

Previous approach (broken): Snapshot all positions → compute ideal layout → teleport. One-shot per frame. No propagation. Other forces overwhelm it.
New approach — spring-like restoring force For each column (sample index p): For each consecutive line pair [i, i+1]: 1. Read currentGap = perpPos[i+1] - perpPos[i] perpPos = (base + matrix) projected onto perpendicular axis 2. Sample midpoint luminance between the two lines midLum = vf.sample(midpoint).lum ^ lumExp 3. Target targetGap = baseGap × lumFactor(midLum) lum=1.0 → compress = baseGap × 0.05 (nearly touching) lum=0.5 → neutral = baseGap × 1.0 (no change) lum=0.0 → expand = baseGap × 5.0 (spread apart) 4. Error error = targetGap − currentGap positive → lines are too close, push apart negative → lines are too far, pull together 5. Apply force = error × magnetStr × mix × dt matrix[i][p] −= force × perpDir × 0.5 (push line i outward) matrix[i+1][p] += force × perpDir × 0.5 (push line i+1 outward) Why this works: • Each pair pushes/pulls toward target gap • Pushing one line changes its gap with OTHER neighbors • Next frame, those gaps generate corrective forces • System settles into equilibrium over ~10-30 frames • Combined with damping: steady-state displacement • Lines never cross because force grows as gap shrinks → stronger push-apart

Damping + Force Equilibrium

The beauty of the continuous model: final displacement = force / damping. This is analogous to terminal velocity. A constant force balanced by proportional drag reaches a steady state.

Equilibrium math

At steady state: force × dt = damping × displacement × dt
Therefore:       displacement = force / damping

Example:
  gradient force   = 0.5 (from Sobel mag × strength)
  damping          = 0.1
  → steady-state   = 5px displacement

  Increase force to 1.0 → displacement drifts to 10px over ~15 frames
  Set force to 0         → displacement decays back to 0 over ~15 frames

Controls:
  Damping slider (0.01 – 1.0)
    Low (0.01)  = slow, drifty, high displacement, takes many frames to settle
    High (0.5)  = snappy, responsive, low displacement, settles in 2-3 frames
    Max (1.0)   = near-instant (approaches v1 teleport behavior)
At high damping (≈1.0) the system behaves like the old instant-displacement model — matrix decays almost fully each frame, forces write near-fresh values. This means the continuous model is a strict superset: you can always get the old behavior back.

Module Reference

Generators

Fixed Angle

  • mix — 0–1 contribution blend
  • amplitude — force magnitude
  • angle — constant direction (degrees)
  • xFactor / yFactor — axis scaling
  • Force: dir × lum × amp × mix × dt

Origin Radial

  • mix — 0–1
  • strength — force magnitude
  • animate — orbit mode, radius, speed
  • Force: unitVec(pt→origin) × lum × str × mix × dt

Gradient Push

  • mix — 0–1
  • strength — force scaling
  • Force: gradVec × mag × str × mix × dt
  • Direction AND magnitude from vector field
  • Sphere: pushes radially inward toward white

Tangent Push

  • mix — 0–1
  • strength — force scaling
  • Force: tangVec × mag × str × mix × dt
  • Perpendicular to gradient = along iso-contours
  • Sphere: pushes in circular arcs

Flow Field

  • mix — 0–1
  • strength — force scaling
  • noise scale — spatial frequency
  • curl — 0=divergent, 1=rotational
  • Force: noiseDir × lum × str × mix × dt

Sine Waves

  • mix — 0–1
  • amplitude — perpendicular displacement force
  • 3 waves — freq + mix each, stacked
  • phase speed — animation rate
  • Force: perpDir × waveSum × amp × mix × dt

Constraints

Magnetism

  • mix — 0–1 restoring force strength
  • compress — gap multiplier at white (0.01–1.0)
  • expand — gap multiplier at black (1.0–8.0)
  • Force per pair: (targetGap − currentGap) × mix × dt
  • Split 50/50 between the two lines in the pair
  • Propagates via neighbor cascade over frames

Boundary

  • mix — 0–1
  • Soft clamp: exponential restoring force near canvas edge
  • Prevents lines drifting off-screen

Modulators

Steer → Gradient

  • mix — 0–1 angular blend
  • Rotates composite vector toward gradient angle
  • Preserves magnitude
  • out = lerp(input, rotated, mix)
  • Only applies where gradient magnitude > dead zone

Steer → Tangent

  • mix — 0–1 angular blend
  • Rotates toward tangent (along contours)
  • Sphere: all forces follow circular paths

Edge Magnitude Scale

  • mix — 0–1
  • At edges (high grad mag): magnitude unchanged
  • At flats (low grad mag): magnitude attenuated
  • out = lerp(in, in × normMag, mix)

Luminance Scale

  • mix — 0–1
  • Bright regions: full displacement
  • Dark regions: attenuated
  • out = lerp(in, in × lum, mix)

Damping

Global Damping

  • damping — 0.01 (drifty) to 1.0 (instant)
  • Applied FIRST each frame before any forces
  • matrix[l][p] *= (1 − damping × dt_norm)
  • At 1.0: matrix nearly zeroes each frame → old behavior
  • At 0.05: slow decay → forces accumulate → fluid settling

Complete Frame Loop

frame(t, dt) { │ ├─ update origin position (if animated) │ ├─ STAGE 1 — DAMP │ for each [l][p]: │ matrix[l][p].fx *= (1 − damping) │ matrix[l][p].fy *= (1 − damping) │ ├─ STAGE 2 — GENERATORS │ for each [l][p]: │ sample vf at base[l][p] │ compute perpN from line geometry │ │ if fixedAngle.mix > 0: │ matrix += dir × lum × amp × mix × dt │ if originRadial.mix > 0: │ matrix += unitVec(→origin) × lum × str × mix × dt │ if gradientPush.mix > 0: │ matrix += gradVec × gradMag × str × mix × dt │ if tangentPush.mix > 0: │ matrix += tanVec × gradMag × str × mix × dt │ if flowField.mix > 0: │ matrix += noiseDir × lum × str × mix × dt │ if sineWaves.mix > 0: │ matrix += perpN × waveVal × amp × mix × dt │ ├─ STAGE 3 — CONSTRAINTS │ if magnetism.mix > 0: │ for each column p: │ for each line pair [i, i+1]: │ currentGap = perpProject(base[i+1]+matrix[i+1]) − perpProject(base[i]+matrix[i]) │ midLum = vf.sample(midpoint).lum │ targetGap = baseGap × lumFactor(midLum, compress, expand) │ error = targetGap − currentGap │ push = error × mix × dt × 0.5 │ matrix[i][p] -= push × perpAxis │ matrix[i+1][p] += push × perpAxis │ ├─ STAGE 4 — MODULATORS │ for each [l][p]: │ {fx,fy} = matrix[l][p] │ if steerGrad.mix > 0: │ steered = rotateToward(fx,fy, gradAngle) │ {fx,fy} = lerp({fx,fy}, steered, mix) │ if steerTang.mix > 0: │ steered = rotateToward(fx,fy, tangAngle) │ {fx,fy} = lerp({fx,fy}, steered, mix) │ if edgeMagScale.mix > 0: │ {fx,fy} = lerp({fx,fy}, {fx,fy}×normMag, mix) │ if lumScale.mix > 0: │ {fx,fy} = lerp({fx,fy}, {fx,fy}×lum, mix) │ matrix[l][p] = {fx,fy} │ ├─ STAGE 5 — RENDER │ for each line l: │ beginShape() │ for each point p: │ colour from lum │ curveVertex(base[l][p] + matrix[l][p]) │ endShape() │ ↻ matrix persists to next frame }

Image-to-Image Distortion

The same matrix pipeline works for distorting images through other images. The vector field from Image A becomes the displacement map for Image B:

Image A → VectorFieldMap → DisplacementMatrix pipeline → UV Offset Map For each output pixel (x, y): offset = matrix[y][x] ← same pipeline output, grid instead of lines sourceXY = (x + offset.fx, y + offset.fy) outputPixel = ImageB.sample(sourceXY) The generators, modulators, and constraints all work identically. The only difference: instead of displacing line vertices, we displace pixel lookups.
